cleaning_services Phase 1 — Text Preprocessing Pipeline

Cleaning & Tokenizing Raw Text

Before our machine learning model evaluates any article, the input text goes through a deterministic 7-step Natural Language Processing (NLP) pipeline built with NLTK to extract high-signal terms.

  • check_circle Cleans HTML tags, URLs, and residual markdown.
  • check_circle Normalizes all text to lowercase.
  • check_circle Splits raw paragraphs into single-word tokens.
  • check_circle Discards punctuation, special symbols, and standard English stopwords.
  • check_circle Lemmatizes terms to their root dictionary form (e.g., "soaring" → "soar").
[INPUT TEXT]
"Tesla stock soars 15% following record quarterly earnings report..."
[NLP PROCESSED TOKENS]

tag Phase 2 — TF-IDF Vectorization

Converting Words into Features

Machine learning algorithms require numerical matrices to compute boundaries. We utilize Term Frequency-Inverse Document Frequency (TF-IDF) with sublinear scaling to build our high-dimensional mathematical space.

  • check_circle Maps the vocabulary space to 50,000 distinct dimensions.
  • check_circle Combines unigrams and bigrams (e.g. "artificial" + "intelligence").
  • check_circle Balances term frequency relative to total article occurrences.
  • check_circle Penalizes extremely common generic words while highlighting rare topic keywords.
[VECTORIZER SETTINGS]
Max Features: 50,000
N-gram Range: (1, 2)
Sublinear TF: True
Norm: L2 Norm
leaderboard Phase 3 — Support Vector Machine

Calibrated Maximum Margin

Out of four distinct algorithms trained, the Support Vector Machine (LinearSVC) achieved the best generalization. We wrap it in a Platt Scaling sigmoid calibrator to extract real-time confidence percentages.

  • check_circle Separation of sparse feature vectors using hyperplanes.
  • check_circle Platt Calibration converts distance-to-margin into probability.
  • check_circle Hyperparameter-tuned using L2 regularization penalty.
  • check_circle Sub-millisecond (0.8ms) inference latency.
[ACCURACY COMPARISON]

dns Phase 4 — Balanced Corpus

Robust & Balanced Dataset

To ensure zero bias across topics, the model was trained on exactly 7,500 articles. We balanced global standard datasets with Nigerian-specific news contexts.

  • check_circle 1,500 balanced entries for each of the 5 topical classes.
  • check_circle Integrates AG News corpus with Nigerian local journalism content.
  • check_circle 85% / 15% train/test stratified split.
  • check_circle Stratification guards against class distribution leakage.
